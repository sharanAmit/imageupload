# AWS S3 & CloudFront Migration Guide

This guide details how to transition the **Trip Memories Platform** storage provider from the local filesystem (`LocalStorageService`) to **Amazon S3** and **Amazon CloudFront** for media delivery.

Because the system was designed using the **StorageService** abstraction and clean architecture, you can perform this migration **without modifying any API routes or frontend scripts**.

---

## Step 1: Create Amazon S3 Bucket & CloudFront Distribution

1. **Create an S3 Bucket**:
   - Create a private bucket (e.g. `trip-memories-vault`).
   - Enable bucket versioning (recommended for production).
   - Keep "Block public access" turned **ON** since we will secure the content.

2. **Configure Origin Access Control (OAC) with CloudFront**:
   - Create a CloudFront Distribution.
   - Set the Origin to your S3 bucket.
   - Configure **Origin Access Control (OAC)** to permit CloudFront to access S3 while blocking direct public S3 requests.
   - Save the S3 bucket policy generated by CloudFront and apply it to your S3 bucket permissions.

## Step 3: Implement S3StorageService

Create a new file `backend/app/services/s3_storage.py` (or add it directly to `storage.py`) containing the S3 service implementation.

```python
import boto3
from botocore.exceptions import ClientError
from fastapi import UploadFile
from backend.app.config import settings
from backend.app.services.storage import StorageService

class S3StorageService(StorageService):
    def __init__(self):
        # Initialize boto3 client
        self.s3_client = boto3.client(
            "s3",
            aws_access_key_id=settings.AWS_ACCESS_KEY_ID,
            aws_secret_access_key=settings.AWS_SECRET_ACCESS_KEY,
            region_name=settings.AWS_REGION
        )
        self.bucket_name = settings.AWS_S3_BUCKET
        self.cloudfront_domain = settings.CLOUDFRONT_DOMAIN

    def upload(self, file: UploadFile, storage_key: str) -> str:
        try:
            self.s3_client.upload_fileobj(
                file.file,
                self.bucket_name,
                storage_key,
                ExtraArgs={"ContentType": file.content_type}
            )
            return storage_key
        except ClientError as e:
            raise Exception(f"S3 Upload failed: {str(e)}")

    def delete(self, storage_key: str) -> bool:
        try:
            self.s3_client.delete_object(Bucket=self.bucket_name, Key=storage_key)
            return True
        except ClientError:
            return False

    def get_url(self, storage_key: str) -> str:
        # Instead of S3 public link, we route traffic through Amazon CloudFront
        # E.g. https://d111111abcdef8.cloudfront.net/trips/trip_uuid/photos/img.jpg
        return f"https://{self.cloudfront_domain}/{storage_key}"

    def get_file_path(self, storage_key: str) -> str:
        # Since it is stored in S3, there is no local path.
        # If needed for direct binary downloads inside Python, we can generate a Presigned S3 URL
        # or stream directly from boto3.
        return self.s3_client.generate_presigned_url(
            'get_object',
            Params={'Bucket': self.bucket_name, 'Key': storage_key},
            ExpiresIn=3600
        )
```

## Step 4: Configure Settings & Swap Dependency Provider

1. Add S3 configuration environment variables in `.env` and `config.py`:
   - `AWS_ACCESS_KEY_ID`
   - `AWS_SECRET_ACCESS_KEY`
   - `AWS_REGION`
   - `AWS_S3_BUCKET`
   - `CLOUDFRONT_DOMAIN`
   - `USE_S3_STORAGE=true`

2. Swap the provider logic in `backend/app/services/storage.py`:

```diff
def get_storage_service() -> StorageService:
-   return LocalStorageService()
+   if settings.USE_S3_STORAGE:
+       from backend.app.services.s3_storage import S3StorageService
+       return S3StorageService()
+   return LocalStorageService()
```

That's it! Once enabled:
- Uploaded media streams directly to your Amazon S3 bucket.
- Gallery cards read images and videos via the high-performance Amazon CloudFront CDN distribution.
- No modifications are required in backend routers (`routes/media.py`) or frontend files (`api.js` or `trips.js`).
